`URISupport.sanitizeUri()` only redacts query-parameter values whose names match a known sensitive-keyword list (password, token, secret, etc.). It does not process the userinfo portion of the URI authority. A typical SMB endpoint URI is constructed as `smb://username:password@host/sharename` — the password in the `user:pass@host` segment will pass through `sanitizeUri` unchanged and appear in the warn log. The PR's stated goal is to mask credentials before logging, so this gap means the goal is only partially met for the most common SMB credential placement. The fix is still strictly better than the original bare concatenation, but callers who embed the password in the authority block remain exposed.
