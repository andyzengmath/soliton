:red_circle: [security] New JSON route dump emits raw `getEndpointUri()` to log and on-disk file with no sanitization in `core/camel-core-engine/src/main/java/org/apache/camel/impl/DefaultDumpRoutesStrategy.java`:216 (confidence: 85)
The new `doDumpStructureJSon` builds a JSON object whose `"from"` field is populated by `def.getInput().getEndpointUri()` directly with no sanitization. Pre-existing XML/YAML paths route URIs through `ModelToXMLDumper`/`ModelToYAMLDumper` which can honor the `setMask` flag (mask=true masks sensitive query params during model→DSL serialization). The new JSON path **bypasses both dumpers entirely** and writes the raw URI to (a) the configured log via `appendLogDump` AND (b) a persisted file via `doDumpToDirectory`. Concrete attack scenarios: `jdbc:mysql://user:passw0rd@host/db?...` (basic-auth credential), `https://api.host/?token=ghp_...` (OAuth), `aws2-s3://bucket?accessKey=AKIA...&secretKey=...`, `kafka:topic?saslJaasConfig=...password=...`, `salesforce:...?refreshToken=...`. CWE-200, CWE-532, OWASP A09 (Logging Failures), OWASP A02 (Cryptographic Failures — credentials in cleartext at rest).
```suggestion
String fromUri = def.getInput().getEndpointUri();
if (mask) {
    fromUri = URISupport.sanitizeUri(fromUri);
}
jo.put("from", fromUri);
```

## Improvements

:yellow_circle: [correctness] Case-sensitive `"json".equals(format)` inconsistent with case-insensitive yaml/xml dispatch in `core/camel-core-engine/src/main/java/org/apache/camel/impl/DefaultDumpRoutesStrategy.java`:160 (confidence: 88)

:yellow_circle: [correctness] `doDumpRoutesStructureAsJSon` ignores the `include` filter unlike XML/YAML paths (confidence: 82)
XML/YAML gate on `include.contains("*") || include.contains("all") || include.contains("routes")`. JSON path always dumps regardless of include. Behavioral inconsistency.

A reader of the new JSON-specific paragraph won't see the "not recommended for production" warning that's two sentences earlier and phrased about XML/YAML.
