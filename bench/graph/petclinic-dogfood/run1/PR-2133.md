## Critical
:red_circle: [correctness] Build target raised from Java 17 to 25 by removing `--release 17` flag in pom.xml:22 (confidence: 92)
The PR drops both `<maven.compiler.release>17</maven.compiler.release>` (pom.xml line 22 in pre-image) and `options.release = 17` (build.gradle:81). Combined with `<java.version>25</java.version>` left in place, the compiler will now target JDK 25 bytecode (major version 69). Any deployment running on Java < 25 ‚Äî including the LTS JDK 17 / JDK 21 commonly used in production ‚Äî will fail at startup with `java.lang.UnsupportedClassVersionError`. The PR description does not mention dropping JDK 17 runtime support. The build-time JDK requirement (Java 22+ for Spring nullability) and the runtime target are independent concerns; only the latter is being changed here, and the change is silent. Maintainer @snicoll flagged this same concern in a PR comment ("I am going to revert the removal of `--release` on 17. While we obviously recommend the latest LTS, there's no reason to prevent users to run petclinic on an older, supported, JDK"), confirming the impact is real and unintentional.
```suggestion
    <java.version>25</java.version>
    <maven.compiler.release>17</maven.compiler.release>
```

## Improvements
:yellow_circle: [correctness] Raw-type regression on `MySQLContainer` in src/test/java/org/springframework/samples/petclinic/MySqlIntegrationTests.java:50 (confidence: 86)
The package move `org.testcontainers.containers.MySQLContainer` -> `org.testcontainers.mysql.MySQLContainer` is fine, but the change also drops the wildcard parameterization: `MySQLContainer<?> container = new MySQLContainer<>(...)` became `MySQLContainer container = new MySQLContainer(...)`. The new class in `testcontainers-mysql` retains the self-bounded generic form `MySQLContainer<SELF extends MySQLContainer<SELF>>` (standard pattern for fluent builders across all Testcontainers `*Container` types). Using the raw form loses type-safe method chaining (`.withUsername(...)` etc. now return raw `MySQLContainer`) and produces an unchecked-warning under `-Xlint:rawtypes`. Same regression applies to MysqlTestApplication.java:38-39. The PR description claims "Replaced deprecated MySQLContainer", but the old class was modularized, not deprecated ‚Äî keeping the parameterization across the package move is the safe migration.
```suggestion
	@ServiceConnection
	@Container
	static MySQLContainer<?> container = new MySQLContainer<>(DockerImageName.parse("mysql:9.5"));
```
